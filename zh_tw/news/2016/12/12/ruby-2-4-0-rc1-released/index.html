Ruby 2.4.0-rc1 發佈

由 naruse 發表於 2016-12-12
翻譯： Juanito Fatas

我們很高興宣佈 2.4.0-rc1 發佈了。

Ruby 2.4.0-rc1 是 Ruby 2.4.0 的第一個候選版本。 為了獲得社群的寶貴意見所以發佈了 rc1。 請不吝至此反饋，您仍可以對將來的功能做出改變。

Vladimir Makarov 改良了哈希表

透過引入 Open addressing 及包含順序的陣列來改善哈希表（st_table）的內部結構。這個改良經過多方論證，特別感謝 Yura Sokolov。

Binding#irb：在 irb 像 binding.pry 一樣啟動一個 REPL

當您在除錯時，您可能常常使用 p 來查看變數的數值。有了 pry 以後，只要在應用裡面寫 binding.pry，就可以啟動一個 REPL 來執行任何 Ruby 程式進行除錯。r56624 補丁給 irb 引入了類似的功能： binding.irb。

Fixnum 和 Bignum 統整為 Integer

雖然 ISO/IEC 30170:2012 沒有明確指定 Integer 類別的細節，但 CRuby 有兩個 Integer 類別：Fixnum 和 Bignum。 Ruby 2.4 統整為 Integer。所有有關 Fixnum 與 Bignum 的 C 語言擴充需要做出修正。

也請參考這個議題和 akr 的簡報。

字串支持 Unicode 的大小寫轉換

String/Symbol#upcase/downcase/swapcase/capitalize(!) 除 ASCII 之外，現在也可以做 Unicode 的大小寫轉換了。

效能優化

Ruby 2.4 還包括以下效能優化及語法變更：

Array#max, Array#min

優化了 [x, y].max 和 [x, y].min，某些情況下不會產生暫時性陣列。

Regexp#match?

新增 Regexp#match? 方法，只做正則匹配，而不會產生參照物件及修改 $~，減少物件的分配。

其他效能改善

  • 提昇實體變量的訪問速度

除錯

Thread#report_on_exception 和 Thread.report_on_exception

除非有另外的線程，明確地與執行中的線程進行 join，否則 Ruby 會忽略執行中線程的異常。啟用 report_on_exception = true 來觀察線程是否有未處理的異常而終止執行。

敬請給我們關於 report_on_exception 預設值以及 report-on-GC（會在線程尚未 join 前被 GC 時給出報告）的建議。

線程死鎖檢查現在會顯示線程的錯誤和相依線程

Ruby 在等待線程執行時會進行死鎖檢查，但檢查結果沒有足夠的資訊來除錯。 Ruby 2.4 的死鎖檢查會顯示錯誤資訊及相依的線程。

請嘗試並享受與 Ruby 2.4.0-rc1 的編碼時光，有任何問題敬請不吝指出！

自 2.3 起重要的變化

  • 支援 OpenSSL 1.1.0 版本
  • ext/tk 從標準函式庫移除 Feature #8539
  • XMLRPC 從標準函式庫移除 Feature #12160

請參閱 NEWS 和 ChangeLog 來進一步了解。

以上變化自 Ruby 2.3.0 以來，計有 2519 檔案變更，程式碼新增 288606 行，移除 83896 行！
